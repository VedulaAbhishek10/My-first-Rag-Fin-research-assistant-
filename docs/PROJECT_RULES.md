Rules

- Never rewrite unrelated files.

- Never delete working code.

- Never continue to the next milestone without approval.

- Prefer readable code over clever code.

- Explain every important design decision.

- Use type hints.

- Keep functions under roughly 50 lines when practical.

- Every public function should have a docstring.

- Generate tests for every module.

- Update README whenever new functionality is added.

- Update architecture diagrams whenever architecture changes.

- Explain all third-party libraries before using them.

- If there are multiple approaches, explain the trade-offs before implementing one.
